# Идея проекта

Цель - собрать учебный RAG, который отвечает на вопросы по открытым литературным текстам и показывает, из каких фрагментов взят ответ.

## Проблема

Без retrieval модель или человек может отвечать по памяти и смешивать факты из разных произведений. В домашнем задании важно показать весь путь: от данных до ответа с источниками.

## Решение

Система индексирует абзацы из Project Gutenberg, режет их на чанки, строит TF-IDF индекс и возвращает top-k фрагментов для вопроса. Генератор ответа не использует внешнюю LLM: он выбирает релевантные предложения из найденных чанков и добавляет ссылки на источники.

## Критерий готовности

- Есть `datasets.json` с 1000+ текстовыми записями.
- Команда `uv run python scripts/build_index.py` строит индекс.
- Streamlit UI показывает ответ и источники.
- Negative-вопрос дает отказ без выдумок.
